GENAI-RON叢書 003

生成AIのしくみ 超詳解

実行構造と責任境界

ユーザーが一文を送ってから、生成AIシステムは何を組み立て、何を参照し、何を実行し、どこで止まり、誰が責任を持つのか。

文・編集責任：Taku

このページは、生成AIのしくみ 超詳解 01〜08の公開版索引である。

このシリーズは、生成AIを「なんとなく賢いチャットボット」としてではなく、request、instruction、context、tool、schema、memory、agentic loop、architectureの組み合わせとして理解するための連続講義である。

中心にある問いは、これである。

ユーザーが一文を送ってから、生成AIシステムは何を組み立て、何を参照し、何を実行し、どこで止まり、誰が責任を持つのか。

1. 全体の読み方

このシリーズは、順番に読むことで、生成AIの実行構造が下から積み上がるように設計している。

01 request / prompt compilation
  ↓
02 instruction hierarchy
  ↓
03 context / memory / retrieval / files
  ↓
04 tool call / tool result / execution loop
  ↓
05 structured output / schema / validation
  ↓
06 memory / personalization / audit
  ↓
07 agentic loop / stop conditions / verification
  ↓
08 full architecture / responsibility boundary

01〜05では、単発の応答がどのように組み立てられるかを見る。

06では、過去情報がどのように保存・参照・注入されるかを見る。

07では、agentがなぜ動き続けるのではなく、止まり、検証し、handoffする必要があるかを見る。

08では、これらをmodel / application / runtime / humanの責任分担として統合する。

2. 公開版リンク

01｜プロンプトは実行時にどうコンパイルされるのか

生成AIのしくみ 超詳解 01｜プロンプトは実行時にどうコンパイルされるのか｜公開版

ユーザーがチャット欄に書いた文章だけがpromptではない。system / developer instruction、履歴、memory、ファイル、検索結果、tool定義などが、実行時に入力環境として組み立てられる。

02｜指示階層とは何か

生成AIのしくみ 超詳解 02｜指示階層とは何か：system / developer / user / tool の権限構造｜公開版

指示の強さは、言葉の強さではなく、置かれた層によって決まる。system、developer、user、tool resultを分け、prompt injectionを「資料と指示の境界事故」として理解する。

03｜contextとは何か

生成AIのしくみ 超詳解 03｜contextとは何か：履歴・メモリ・検索結果・ファイルはどう入力へ組み込まれるのか｜公開版

contextとは、保存情報そのものではなく、今回の生成に注入された入力集合である。履歴、memory、retrieval、file input、tool resultがどのように現在の応答に影響するかを見る。

04｜ツール実行ループとは何か

生成AIのしくみ 超詳解 04｜ツール実行ループとは何か：tool callはどのように発生し、実行され、戻るのか｜公開版

modelはtoolを直接実行しない。modelがtool callを出し、applicationが検証・実行し、tool resultをcontextへ戻す。外部世界への接続がどの層で起きるかを扱う。

05｜構造化出力とは何か

生成AIのしくみ 超詳解 05｜構造化出力とは何か：JSON、schema、validation、再生成｜公開版

構造化出力とは、きれいなJSONを書かせることではない。schema、parse、validate、refusal、retry、repair、handoffまで含む、applicationとの契約として扱う。

06｜memoryとは何か

生成AIのしくみ 超詳解 06｜memoryとは何か：保存、参照、personalization、監査｜公開版

memoryとは、モデル内部に人間の記憶のような体験が発生することではない。保存された情報が選別され、現在のcontextへ再注入される仕組みとして理解する。

07｜停止条件と検証ループ

生成AIのしくみ 超詳解 07｜停止条件と検証ループ：agentはいつ続行し、いつ止まるのか｜公開版

agentic loopとは、モデルが自律的に動き続けることではない。停止条件、budget、verification、guardrails、handoff、traceによって制御されるloopである。

08｜全体アーキテクチャ

生成AIのしくみ 超詳解 08｜全体アーキテクチャ：model / application / runtime / human の分担｜公開版

生成AIシステムはmodelだけでは成立しない。model、application、runtime、memory、tools、guardrails、human、auditの責任境界を明示する全体設計として統合する。
