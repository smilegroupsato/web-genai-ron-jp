1. 生成AIシステムはmodelだけではない

日常的には、私たちは「AIが答えた」「AIが検索した」「AIがファイルを読んだ」「AIがNotionにページを作った」と言う。日常語としてはそれでよい。しかし、しくみとして見るなら、この言い方はかなり粗い。

モデルが直接Webを検索しているとは限らない。モデルが直接ファイルシステムを読んでいるとは限らない。モデルが直接メールを送っているとは限らない。多くの場合、モデルがしているのは、自然文、構造化出力、tool call候補、拒否、確認依頼、handoff提案などを生成することである。

"AI did it"
  = model generated something
  + application interpreted it
  + runtime executed something
  + tool returned a result
  + application verified or displayed it
  + human accepted, corrected, or approved it

2. 四つの大きな層

層主な責務混同してはいけないこと
modelcontextをもとに、自然文、構造化出力、tool call候補などを生成する。外部世界を直接操作していると思い込む。
applicationrequest、instruction、context、memory、tool、validation、loopを編成する。modelの自然文応答をそのまま真実や完了扱いする。
runtimetool、API、connector、file system、database、sandboxなどを実行する。実行成功をtask成功と同一視する。
human目的、判断、承認、責任、評価、意味づけを担う。すべてを自動化できるとみなす。
AI system ≠ model alone

AI system =
  model
  + application orchestration
  + context assembly
  + tool runtime
  + memory / retrieval
  + validation / verification
  + guardrails
  + human judgment
  + audit trail

3. requestからresponseまでのpipeline

1. user intent
2. request assembly
3. instruction hierarchy placement
4. context selection
5. memory / retrieval injection
6. tool schema exposure
7. model invocation
8. output parsing
9. validation / refusal handling
10. tool execution or direct response
11. result observation
12. verification
13. loop decision
14. final response / artifact / handoff / stop
15. trace / audit / evaluation

これは単なる実装手順ではない。責任境界の一覧である。どの段階で何を確認するのか、どの段階で人間へ返すのか、どの段階でtoolを許可するのか、どの段階で止めるのか、どの段階を記録するのかを分けるための地図である。

4. model layerの責務と限界

model layerの責務は、与えられたcontextから次の出力を生成することである。answer text、structured output、tool call candidate、classification、plan、refusal、clarification request、handoff proposalなどを生成する。

model output ≠ external fact
model output ≠ tool execution
model output ≠ permission
model output ≠ user approval
model output ≠ completed task

modelはsource of truthではない。memory storeでもない。外部世界を直接操作する主体でもない。責任主体でもない。だから、model outputとapplication actionを分ける必要がある。

5. applicationとruntimeの責務

application layerは、modelを呼び出すだけの薄い殻ではない。何をcontextへ入れ、どのtoolを見せ、model出力をどうparseし、何をvalid outputとみなし、どこで人間へ戻すかを決める編成層である。

runtime / tool layerは、外部世界への実行境界である。toolは単なる能力追加ではなく、permission boundary、side effect boundary、audit boundaryを含む。

6. humanは外側ではなく構成要素である

human layerは、目的、判断、承認、責任、評価、意味づけを担う。生成AIシステムから人間を外すのではなく、どこで人間に戻すのか、どこまで委任するのか、どの判断は保持するのかを設計する必要がある。

中心フレーズ

生成AIシステムとは、model、application、runtime、humanの責任境界を持つ複合システムである。
