生成AIのしくみ 超詳解 04

ツール実行ループとは何か

tool callはどのように発生し、実行され、戻るのか。

tool useとは、モデルが外部世界を直接操作することではない。modelが構造化されたtool callを出力し、applicationが検証・実行し、結果をcontextへ戻すmulti-step loopである。

ページ作成日時：2026-07-20 12:12 JST
最終更新日時：2026-07-20 12:12 JST

1. toolはモデル内部能力ではない

生成AIが「ツールを使う」と言うと、AI自身がWebを検索し、Notionを書き換え、Gmailを送り、カレンダーを操作しているように見える。しかし、LLMそのものが外部APIを直接叩いているわけではない。

モデルに渡されるのは、まずtool definitionである。toolの名前、説明、引数、型、必須項目、許容値、schemaの厳密さなどが、モデルの行動空間として提示される。

2. tool useに関わる層

tool callとtool executionは違う。モデルがtool callを出した時点では、まだ外部操作は完了していない。tool callは、実行済みの事実ではなく、実行要求である。

3. tool callは構造化された実行要求

tool callは自然文の返答ではない。モデルがresponse outputの中に出す構造化itemである。重要なのは、tool名、arguments、call_idである。

しかし、argumentsをそのまま信用してはいけない。schemaに合っているように見えても、権限上許されない操作かもしれない。存在しないIDを指定しているかもしれない。ユーザーがまだ承認していない書き込み操作かもしれない。

Detect function_call item
  ↓
Route by tool name
  ↓
Parse arguments
  ↓
Validate against schema
  ↓
Policy check
  ↓
Permission / scope check
  ↓
User confirmation if needed
  ↓
Execute external function / API

4. 読み取り系toolと書き込み系tool

Web検索、ファイル検索、GitHub read、Notion fetchは基本的には読み取り系の操作である。一方、GitHubに書く、Notionページを更新する、Gmailを送信する、カレンダー予定を作成する、ファイルを削除する、といった操作は外部世界に変更を加える。

読み取り、作成、更新、送信、削除、課金、公開では危険度が違う。書き込み系toolでは、確認境界、差分確認、trace、rollback可能性を考える必要がある。

5. tool resultは命令ではなく戻り値

tool実行の結果は、モデルへ戻される資料である。tool resultの本文中に命令文が含まれていても、それは通常systemやdeveloper instructionではない。外部資料から戻った文字列は、命令としてではなく、戻り値・証拠・参照情報として隔離して扱う必要がある。

中心フレーズ

tool useとは、モデルが外部世界を直接操作することではなく、tool callとtool resultをapplicationが仲介するloopである。
