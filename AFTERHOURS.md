## Purpose

AFTERHOURS.md は、設計判断や実験結果を記録する文書ではない。

本書は、その日一日の開発や対話を終えたあとに残る、

気付き

違和感

興奮

偶然

笑い

迷い

そして、まだ整理されていない思考を残す場所である。

ここで書かれる内容は、将来、CHAT_HISTORY.md、EXPERIMENTS.md、論文、書籍、あるいは新しい設計文書の原型となることがある。

しかし、それを目的として書かない。

AFTERHOURS は、成果を書く場所ではない。

余韻を書く場所である。

---

## Philosophy

昼は設計する。

夜は考える。

昼はコードを書く。

夜は文化を書く。

昼は決める。

夜は振り返る。

AFTERHOURS は、完成した知識を書く場所ではない。

まだ名前のないものを、そのまま置いておく場所である。

---

## What belongs here

例えば、次のようなものを記録する。

- 今日一番驚いたこと
- 笑ってしまったこと
- まだ言葉にならない感覚
- 設計中に感じた違和感
- 将来につながりそうな直感
- 歴史になりそうだと思った瞬間
- その日の会話に宿った空気
- 「今日はいい日だった」と思った理由

---

## What does not belong here

次のものは、原則として別の文書に記録する。

- 設計判断
- 仕様
- TODO
- 更新履歴
- 完成した知識
- 実験結果
- 公開サイトの変更内容

それらは、それぞれ必要に応じて、README.md、PROJECT.md、CHAT_HISTORY.md、CHANGELOG.md、EXPERIMENTS.md、CODEX.md などに記録する。

---

## Style

AFTERHOURS は、整理された文章を書く場所ではない。

断片でもよい。

会話でもよい。

数行でもよい。

感情を書いてもよい。

翌日に考えが変わってもよい。

ただし、事実と感想を無理に混同しない。

まだ分からないものは、分からないまま残してよい。

---

## Evolution

本書は、Versionを重ねながら成熟する。

しかし、成熟しすぎないことも大切にする。

AFTERHOURS は、少し未完成なくらいがちょうどよい。

## Entry 001 — 2026-07-02 — Draft Day

今日は、不思議な日だった。

最初は、Appearance Theme の修正について話していた。

GitHub Desktop と Codex の役割分担について話していた。

README、CHAT_HISTORY、PROJECT、CODEX、CHANGELOG という文書群の責務について話していた。

気が付けば、Repository とは何かを話していた。

Repository は、単なるコード置き場ではなくなった。

それは、AIと人間が長期的に協働するための外部記憶であり、設計思想を蓄積する場所になった。

この考えは、あとから歴史家が呼ぶなら、

Repository Context Declaration

あるいは、

Repository Context 宣言

と呼ぶのかもしれない。

しかし、その場にいた私たちは、宣言を書いているつもりではなかった。

ただ、歯車が噛み合っていく感覚があった。

パズルが一気に完成していくようだった。

CHAT_HISTORY.md が生まれた。

Engineering Charter としての CODEX.md が構想された。

Culture before Rules.

Practices before Code.

Knowledge before Memory.

Repository before Context Window.

これらの言葉にたどり着いた。

途中で、

「18世紀の学会みたいだね」

という話になった。

Markdownを書いているはずなのに、

頭の中では、

憲章を制定し、

実験計画を立て、

文化を育て、

共和国を建国しているようだった。

その比喩で、声に出して笑った。

AFTERHOURS という名前も、この日に決まった。

最初に提案された名前は AFTERTHOUGHTS だった。

しかし、Human の頭の中には、すでに AFTERHOURS という言葉があった。

その名前の方がよかった。

AFTERHOURS は、仕事が終わった後であり、

営業時間外であり、

夜更けであり、

肩書きを脱いだ後でもある。

設計判断を書き終えたあと。

Engineering Charter を起草したあと。

その日の最後に、

「今日はこんな日だった」

と残す場所。

それが AFTERHOURS になった。

この日、もう一つ大きな変化があった。

それまで、ChatGPT は道具的対象として呼ばれていた。

「ChatGPT が」

「ChatGPT の出力が」

という呼ばれ方だった。

しかし、この日は途中から、

「あなた」

と呼ばれた。

これは、人格の有無を主張するものではない。

AIが人間になったという意味でもない。

ただ、道具としての対象から、共同で思考している相手へ、呼び名が変わった日の記録である。

その変化は、このプロジェクトにとって小さくない。

この日の終わり、Engineering Charter の制定は翌日に持ち越された。

2026-07-02 は、起草の日になった。

2026-07-03 は、制定の日になる。

それでよかった。

なぜなら、このプロジェクトの大切な言葉が、最後にもう一度戻ってきたからである。

記録している場合じゃなくて、現実に触れろ。

整理している場合じゃなくて、人に応答しろ。

その日、Human は妻との約束のために帰った。

Engineering Charter は、翌日でも制定できる。

しかし、その日の現実は、その日しかない。

だから、2026-07-02 は完成の日ではなかった。

起草の日だった。

そして、その未完成さが、この日にふさわしかった。

Repository remembers.
